第一章 交换概论

通信网（三要素）
  • 交换设备：交换机
  • 传输设备：用户线、中继线以及其他传输设备
  • 终端设备：用户终端
交换方式的特点
  • 电路交换
    • 通信网最早出现的交换方式，也是应用最普遍的一种交换方式，主要应用于电话通信网中
    • 通信过程
      • 摘机
      • 听到拨号音后拨号
      • 交换机寻找被叫
      • 向被叫振铃同时向主叫送回铃音
        - 此时表明电话网的主被叫之间已经建立起双向的话音传送通路
      • 被叫摘机应答，进入通话阶段
      • 任何一方挂机
      • 交换机拆除已经建立的通话通路
      • 交换机向另一方发送忙音提示挂机
      • 结束通话
    • 电话通信分为三个阶段：呼叫建立、通话、呼叫拆除
    • 电路交换也可以分为：连接建立、信息传送、连接拆除
    • 6个特点
      • 信息传送的最小单元是时隙
        • 每个TS为8bit，通信速率为64kbit/s
        • TS是电路交换传输、复用、交换的最小单位
      • 面向连接的工作方式（物理连接）
        • 电路交换的基本过程是连接建立、信息传送和连接拆除，要在源和目的之间建立信息通路的连接
        • 是一条物理的连接通路，只要通信即刻可以传送信息
        • ==传统固定电话网（PSTN）==
      • ==同步时分复用（固定分配带宽==）
        • 把时间划分成等长的基本单位帧，每个帧再划分成更细的时隙，时隙根据其在帧中的位置编号（比如第10个就编号为9）——位置化信道
        • 那么一个帧携带的n个时隙就相当入一次带“n个人的信息”，而每个时隙中的第i帧组成了第i个信道
        • 电路交换基于PCM30/32路同步时分复用系统，每秒传送8000帧，每帧32个时隙，每个时隙8bit，每路通信信道TS为64kbit/s恒定速率，即面对每路通信所分配的带宽是固定的，即使没有信息传送也会占着信道
      • 信息传送无差错控制
        • 话音业务的特点是实时性要求高，对可靠性要求没有数据通信高，因此电路交换中为了减少话音信息的时延、没有CRC骄阳、重发等差错控制
      • 信息具有透明性
        • 电路交换对于所传送的话音信息不做任何处理而是原封不动的传送——透明传送
      • 基于呼叫损失制的流量控制
        • 过负荷的时候再到的呼叫会直接呼损掉，从而达到流量控制的目的，过负荷时呼损率会增加，但是不影响已经建立的呼叫
    • 通信网采用的交换方式一定要适应其业务性
    • 缺点
      • 电路交换对于数据交换的可靠性没有分组交换高，不适合差错敏感的数据业务
      • 电路交换采用固定带宽分配的方式，电路利用率低，不适合突发业务、只适合实时性、恒定速率的业务
  • 分组交换
    • 将用户要传送的信息分割为若干个分组，每个分组中有一个分组头，含有可供选路的信息以及其他的控制信息
    • 本质就是存储转发，将接收的分组暂时存储下来，在墓地方向路由上排队，当它可以发送消息时再将信息发送到相应的路由上完成转发
    • 两种更具体的分组交换
      • 虚电路VC
        • 采用面向连接的工作方式
        • 通信过程与电路交换类似，具有连接建立、数据传送和连接拆除三个阶段
        • 用户数据传送前先建立端到端的虚连接，连接建立之后属于同一呼叫的数据分组均沿着这一虚连接传送，通信结束后拆除该连接
        • 虚连接=虚电路，是逻辑连接，由交换节点保存选路结果和路由关系
        • 分组头中有对应于建立的VC的逻辑信道标识
        • 传输时延小、抖动小、分组有序到达、网络故障重连
      • 数据包DG
        • 采用无连接工作方式
        • 呼叫前不需要事先建立连接，而是边传信息边选路，并且每个分组一句分组头中的目的地址独立的精心选路
    • 分组交换的特点
      • 信息传送的最小单位是分组
        • 分组由分组头和用户信息组成，分组头具有选路和控制信息
      • 面向连接和无连接两种工作方式
      • ==统计时分复用（动态分配带宽）==
        • 把时间划分成不等长的时间片，长短不同的时间拍呢就是传送不同长度分组所需要的时间，对于每路通信没有固定分配时间片
        • 动态分配带宽，分组多久占的时间片多
        • 识别每路通信的分组需要一句分组头中的标志来区分，也称为信道标识——标识化信道
        • 信道利用率更高
        • 信息传送有差错控制
          • 专门为数据通信网设计的交换方式，特点就是可靠性要求较高
          • 分组交换设有CRC骄阳、重发等差错控制机制
        • 信息传送不具有透明性
          • 分组交换会对于信息有拆分、重组等
        • 基于呼叫延迟制的流量控制
          • 如果数据流量较大，分组会排队处理
    • 分组交换不适合实时要求较高的话音业务，适合突发和对差错敏感的数据业务，在数据通信网中被广泛采用
  • PS：交换重要概念
    • 面向连接和无连接
      • 面向连接工作方式的特点（公交车道）
        • 不管是物理连接还是逻辑连接，通信过程都可以分为：连接建立、传送信息、连接拆除
        • 路由选择和数据转发分开进行
        • 一旦连接建立，所有信息均沿着连接路径传送
        • 可以保证信息的有序性
        • 传送时延更小
        • 一旦连接出现故障，信息传送就要终端，需要重新建立连接（故障敏感）
      • 无连接工作方式的特点（自己开车）
        • 没有连接建立的过程，一边选路一边传送信息同时进行
        • 属于同一个通信的信息沿着不同的路径到达目的地，路径事先无法预知，无法保证信息的有序性
        • 信息传送的时延比面向连接工作方式时延大
        • 对于网络故障不敏感
        • ==IP网为无连接工作方式==
    • 通信带宽分配
      • 固定带宽分配
        • 适用于固定比特率业务（如话音），需要预留资源
      • 动态贷款分配
        • 适用于时变比特率业务，需资源动态分配或竞争
  • 帧交换FS
    • 分组交换无法支持高速的数据通信，协议包含了下三层，并且二层和三层都有差错+流量控制，是的信息通过交换节点的时间增加
    • 帧交换是一种帧方式的承载业务，简化了协议，协议栈只有物理和数据链路层，加快了处理速度
      • 二层上传输的数据单元为帧，也就称为帧交换
  • 帧中继FR
    • 信息传送的最小单位是帧
    • 相比于帧交换协议进一步简化
    • 只保留二层数据链路层的核心功能如帧定界、同步、传输差错检测，删除了流量控制、重发等功能
    • 帧中继/帧交换的前提条件
      • 高带宽、高质量的传输线路（如光纤），使得简化差错控制和流量控制称为可能
      • 终端系统日益智能化，把智能化的纠错功能交给终端来完成
    • 链路层引入数据链路连接标识符DLCI，完成分组交换的虚电路功能
      • 实际的实现有点像MPLS，是在第二层实现了伪路由
  • ATM交换
    • 以分组传送模式为基础并融合了电路传送模式的优点
    • 具体特点
      • 固定长度的信元和简化的信头
        • 信息传送最小单元是信元cell，信元53byte，前5byte是信头，后48byte是信息域
        • 信头简化了虚连接标志、优先级标志、信息类型标志、信头差错检验等部分，简化了交换节点的处理开销，加快了交换的速度
        • ATM只对信头做差错检验，简化了操作
        • 固定长度有利于减少时延抖动，简化硬件交换的实现
      • ==采用了异步时分复用==
        • 动态分配带宽，不固定分配时间片
        • 与统计时分复用的区别是时间片等长
        • 还是通过信头标志来区分是哪路通信的信元
      • 面向连接的工作方式
        • 是逻辑连接（与虚电路类似）
        • 称为虚连接VC，且分为两级
          • 虚通道连接VPC、虚路径标识符VPI
          • 虚信道连接VCC、虚通道标识符VCI
交换技术的种类和特点

各种常用网络的交换方式

  • PSTN：电路交换
  • B_ISDN（宽带综合业务数字网）：ATM交换
  • Internet：分组交换
  • No.7：信令转移点STP
  • GSM：电路交换
  • GPRS：电路+分组
  • 3G：先电路+分组，后纯分组
  • 4G&5G：分组
  • MPLS：带标记的分组
各种常用网络的重要信令/协议
  • PSTN：No.7
  • MPLS：LDP、SIP
  • Internet：IP
  • SDN：Openflow
电信交换系统的基本结构
  • 由信息传送子系统和控制子系统组成
    • 信息传送子系统
      • 包括交换网络和各种接口
        • 交换网络（交换机构）
          • 交换就是信息从某个接口进入交换系统经交换网络的交换从某个接口出去
          • 完成交换功能的主要部件就是交换网络
          • 涉及到交换网络的拓扑结构、交换网络内部选路策略、交换网络的控制机理、多播完似的实现、网络阻塞特性、网络可靠性等互连技术
        • 接口
          • 将进入交换系统的信号转变为交换系统内部适应的信号
          • 变换包括信号码型、速率等方面的变换
          • 接口类型
            • 用户接口：交换机用户线的接口
            • 中继接口：交换机连接中继线的接口，一般为数字中继接口
    • 控制子系统
      • 交换系统的指挥中心
      • 由处理机及其运行的系统软件、应用软件和OAM软件组成
      • 使用信令与用户和其他交换系统进行协调和沟通来完成对交换的控制
        • 信令是通信网中规范化的控制命令，控制通信网中各种通信的连接以及拆除
        • 交换系统和用户交互的信令叫用户信令，交换系统之间交互的信令叫局间信令

第二章 交换网络

  • 交换网络的三个基本要素：交换单元、拓扑连接、控制方式
    、##### 连接概念：
  • 交换机的基本功能就是在入线和出线之间建立一定的连接
  • 各种表示方法
    • 集合描述方式：集合描述法：所有入线序号为集合𝑇，出线序号为集合𝑅。连接c={t,Rₜ},t∈T,Rₜ⊂R
    • 函数描述法：f(t)=Rₜ, Rₜ⊂R
    • 排列表达式 上面入下面对应出，可重复
    • 入线排列表达式：（t ₀，t ₁…t ₙ₋₁）
      • 这里n个位置表示n个出端、不可重复，i号位写几代表几号入端连i号出端
      • 出线排列表达式类似
    • 图形表示：画图、连线
    • 二进制表示，是点到点的连接，左边和右边的二进制编号的端口相连
  • 直线连接I
    • 自己连自己
    • I(x₂x₁x₀)=x₂x₁x₀
  • 交叉连接E
    • 两两一组相连，二进制就是和末尾相反的连
    • E(xₙ₋₁xₙ₋₂···x₁x₀)=xₙ₋₁xₙ₋₂···x₁x₀（取反用下划线代替了上划线很阴间）
  • 蝶式连接β
    • 二进制最高位和最低位互换
    • β(xₙ₋₁xₙ₋₂···x₁x₀)=x₀xₙ₋₂···x₁xₙ₋₁
    • 子蝶k是低k位做蝶式
    • 超蝶k是高k位做蝶式
  • 均匀洗牌连接δ
    • 二进制循环左移一位
    • δ(xₙ₋₁xₙ₋₂···x₁x₀)=xₙ₋₂···x₁x₀xₙ₋₁
    • 子洗牌低位左移
    • 超洗牌高位左移
  • 逆均匀洗牌δ⁻¹
    • 二进制循环右移一位
    • δ(xₙ₋₁xₙ₋₂···x₁x₀)=x₀xₙ₋₁xₙ₋₂···x₁
    • 子洗牌低位右移
    • 超洗牌高位右移
  • 间隔交叉连接（立方体连接）
    • Cᵢ的方体置换指和自己第i位相反的连
    • C ₖ(xₙ₋₁xₙ₋₂···x₁x₀)=x₀xₙ₋₁xₙ₋₂··xₖ··x₁
  • 点对点、同发、广播
    • 如果一个起点对应一个终点——点到点连接
    • 如果一个起点对应多个终点——点到多点连接
      • 多点指部分终点——同发
      • 多点指全部终点——广播
    • 交换单元不允许多对多或多对一的连接，称为出现冲突
交换单元的结构、特性、工作原理
  • 交换单元的分类
    • 有向交换单元和无向交换单元
      • 有向交换单元是单向的，只能入线进出线出
      • 无向交换单元有N无向和KxL无向两种
        • N的话是随便进随便出
        • KxL必须是K进L出或者L进K出
    • 按照交换单元入线和出线的数量关系
      • 分别可以叫：集中器、置换器、扩展器
    • 按照交换单元所有入线和出线之间是否共享单一的通路
      • 时分交换单元的所有输入口和输出口之间共享唯一的通路（共享总线/共享存储器）
      • 空分交换单元入线出线有很多通路
    • 按照交换单元所接受的模拟信号和数字信号
      • 分为数字交换单元和模拟交换单元
  • 基本交换单元与开关阵列
    • 由大量开关组成的阵列称为开关阵列
    • 开关一般位于入线和出线的交叉点、有接通断开两种状态，开关分为单向开关和双向开关
    • MxN有向交换单元
    • N无向交换单元
      • 双向开关
        • 自己和除了自己之外的所有人连
        • 共N（N-1）/2个开关
      • 单向开关
        • 双向的数量翻个倍
    • MxN无向交换单元
    • 如果每个交点都有交换开关，称为全连通交换单元
      • 否则为非全连通交换单元
  • 开关阵列的性质
    • 容易实现同发和广播功能
      • 打开对应的开关即可
    • 信息从入线到出线具有均匀的单位延迟时间
      • 从入线到出线经过的开关数量相等，因此信息延迟也是相等的，不存在时延抖动
    • 开关阵列控制简单
      • 用两值信号即可
    • 开关阵列适合较小规模交换单元
      • 规模大的时候成本太高
    • 开关阵列的性能依赖于使用的开关
    • 同发和广播容易实现
    • 无内部阻塞
  • S接线器（空间接线器）
    • 空间交换单元
    • 用于完成多个输入复用线与多个输出复用线之间的空间交换，而不改变时隙位置
    • 基本结构
      • S接线器由交叉点矩阵和一组控制存储器构成
        • 交叉点矩阵
          • 一般有相同数量的入线和出线
          • NxN的S接线器有N²个交叉点
          • 实际的交叉点矩阵多由选择器构成
            • 这里要注意，输入控制方式指的是输入端口一一匹配，控制信号控制输出端口，别弄反了
        • 控制存储器CM
          • 控制存储器的数量等于输入or输出线的数量
            • 假设输入控制，那么一个存储器对准一个输入线
          • 控制存储器的单元数等于输入or输出线的时隙数
            • 一个存储器的所有时隙单元对应输入线的每个时隙
          • 存储器每个单元的位数要足够表示对应的输出or输入线
            • 比如一共4入4出，单元就要有2bit来表示4个出口/入口
    • 控制方式
      • 输入控制方式
        • CM个数按照输入线的个数配置，一一对应
        • 单元数等于输入复用线时隙数
        • i号CM的j号单元的内容k表示：第i个输入线在第j时隙要与第k个输出线接通（输出之后仍在第j时隙）
        • 输入控制方式只要每个CM的同一个时隙设置在同一个端口就可以多对一（不过没用）
      • 输出控制方式
        • CM个数按照输出线的个数配置，一一对应
        • 单元数等于输出复用线时隙数
        • i号CM的j号单元的内容k表示：第i个输出线在第j时隙要与第k个输入线接通（输出之后仍在第j时隙）
        • 输出控制方式只要每个CM的同一个时隙设置在同一个端口就可以广播（有用）
        • 可以自动避免出现冲突（因为一个时隙一个出口只能填一个入线）
    • 特点
      • 只完成空间交换、不进行时隙交换
      • 空间接线器按照时分的方式工作，输入线和输出线都是时分复用线
      • 严格无阻塞
    • 空间接线器用于构建数字电话交换系统的交换网络，完成对PCM信号的交换
  • 时分交换单元
    • 时分交换单元的一般构成
      • 时分交换单元内部只有一条通路，所有的信息必须通过唯一的通路完成交换
      • 公共通路可以是存储器或者总线，时分交换单元也因此分为共享存储器型交换单元和共享总线型交换单元
        • 共享存储器型交换单元
          • 核心存储器被分为了N个区域，N路输入信号存放在N个区域，不同区域的N路信号被读出组成N路输出信号
          • 输入缓冲方式
            • N个区域和输入信号对应，在适当的时候输出到输出信道
          • 输出缓冲方式
            • N个区域和输出信号对应，不同输入信道来的信息按照要输出信道放到对应的区域内，时刻到来时直接输出
        • 共享总线型交换单元
          • 把总线的工作时间划分成N个时隙，每个时隙把总线分给相应的输入部件，输入部件可以把入线上的信息送到总线上，输出部件会读出总线上的信息输出
          • 输入部件会接收入线信号、进行格式变换、在对应时隙把信息放上总线，输入部件有缓冲存储器，如果输入部件每τ获得一个时隙，输入信号速率为V bit/s，则输入部件容量至少为Vτ bit
          • 输出部件会监测总线信号，将本端口的信息读出，进行格式变换，在出线输出
          • 公式
            • N条入线、每条入线的信号速率为V、总线上信号速率为NV
            • 总线的一个时隙长为T，一个时隙只能传B个bit
            • k为时隙分配规则因子，随着时隙规则复杂而减小
            • 可以通过增加B、减小T、减少k来来增加交换单元容量（NV）
              • 增加B——增加总线
              • 减小T——用存储快速存储器
  • T接线器（时间接线器）
    • 典型的共享存储器型交换单元
    • 输入——入复用线，输出——出复用线
    • 基本结构
      • 话音存储器SM
        • 存储话音信息
        • 存储单元个数和输入/输出的时隙相等
        • 时隙只传输8bit
        • 大小为时隙数x8 bit
      • 控制存储器CM
        • 保存的是对应SM单元该输出/写入的时隙
        • 存储单元个数与SM相等，一一对应
        • 单元位数要能表示时隙的个数
          • n个时隙就log₂ⁿ位
        • 大小为时隙数xlog₂ⁿ bit
    • 控制方式（例子：第k时隙的信息a需要在第i时隙出）
      • 输出控制方式——顺序写入、控制读出
        • 输入线的信息按照时隙号顺序写入相应的单元（直接进第k单元写下a）
        • 控制读出，因此控制器按照输出的顺序排列，第几单元代表第几个时隙出（在第i单元写下k代表i时隙输出SMk中的a）
        • 读出的时候按照控制器一个个往下找对应的SM读出
        • 有概率会延迟到下一帧的对应时隙输出
      • 输入控制方式——控制写入、顺序读出
        • 输入线的信息要控制好输入的单元，第i时隙出就乖乖输入到第i单元，写下a
        • 控制读入，因此控制器按照输入的顺序排列第几单元代表第几个时隙入（在第k单元写下i代表k时隙输入SMi中的a）
        • 读出是直接顺序从SM中读出即可
    • 注意的点
      • 时间接线器的控制存储器是由控制单元写入程序，控制存储器相当于信息交换的交换控制表
      • 话音存储器SM需要在一个时隙内完成一次读操作和一次写操作，控制存储器也要完成一次读操作和写操作，访问速度必须支持两个存储器个完成一次读写操作
      • 经过时间接线器交换的信息存在时延（想传到已经过去了的前面的时隙）
        • TS0的读和TS0的出会在一个时隙的前半和后半周期同时进行，所以如果i时隙进要换到i时隙恰好就可以无缝出，但如果i进i-1出，那此时已经过了i-1的出的时间，得这一帧都走完了轮到下一帧的i-1时隙才能出了
无阻塞网络概念
  • 交换网络的阻塞是指从交换网络不同输入端来的信息在交换网络中交换时发生了对同一公共资源争抢的情况
    • 竞争失败的信息会被阻塞直到公共资源重新释放
  • 竞争的分类：内部竞争和出线竞争
    • 内部竞争：两路信息同抢交换网络内部某一条通路
    • 出线竞争：不同入端争抢同一个输出端口
  • 阻塞和无阻塞
    • 存在内部阻塞的交换网络称为有阻塞
    • 不存在内部阻塞的交换网络称为无阻塞，无阻塞哈会有三种细分含义：
      • 严格无阻塞：只要连接的起点和终点是空闲的，任何时候都可以在交换网络中建立一个连接
        • 不需要改变现有连接
      • 可重排无阻塞：只要连接的起点和终点是空闲的，任何时候都可以直接或者间接对已有连接重新选路来建立一个连接
        • 需要改变已有连接腾地方
      • 广义无阻塞：在顺序建立连接时遵循一定规则的前提下，只要连接的起点和终点是空闲的，任何时候都可以在交换网络中建立一个连接
交换网络
  • 基本结构
    • 交换单元按照一定的拓扑结构扩展而成
    • 输入端和输出端称为入线和出线
    • M条入线和N条出线称为MxN交换网络
  • 交换网络分类
    • 单级交换网络和多级交换网络
      • 单级交换网络：
        • 由一个或者多个位于同一级交换单元所构成的交换玩络
        • 本质就是交换信息从入到出只经过一个交换单元
        • 如果同一级由多个交换单元构成时，不同交换单元的入线与出线间可建立连接
          • 不用过度纠结，就是可以交叉连接的意思，代表不是直连
      • 多级机交换网络（MIN）
        • 信息从输入端到输出端会经历多个交换单元
        • 根据多级互连网络分为k级（层次分明）
          • 交换网络输入端只连到第一级的入线
          • 第一级的出线连到第二级的入线
          • 到第k-1级的出线只连到第k级的入线
          • 第k级的出线连到交换网络输出端
    • 单通道交换网络和多通路交换网络
      • 单通道交换网络：
        • 任何一个入线与一个出线之间只存在唯一的通路，没有选择
      • 多通路交换网络
        • 任意一个入线与出线之间有多条通路（那么也可以进行重排）
    • 时分交换网络与空分交换网络
      • 时分交换网络
        • 所有输入和输出端口分时共享单一的通信通路，具有时隙交换功能
      • 空分交换网络
        • 可以在多对输入端口与输出端口间同时并行的传送信息
        • CLOS网络与banyan网络属于典型的空分交换网络
      • 时空结合网络
        • 既能完成时隙交换也能完成空间交换
        • 如TST网络和DSN网络
  • CLOS网络
    • 思想：采用较小规模的交换单元按照某种连接方式连接起来从而构成多级交换网络
    • 如果入线M=出线N，称为对称的CLOS网络，否则为非对称的CLOS网络，无说明默认CLOS网络对称
    • 经典3级CLOS网络
      • 构造法
        • 入线N被分为了r组，每组n条入线
          • N=rxn
        • 第一级有r个nxm交换单元
        • 第二级有m个rxr交换单元（m可人为设定）
          • 第一级的第i个单元的m条线分别连接第二级m个单元的第i个口（按照顺序分法）
        • 第三级和第一级对称
      • 严格无阻塞条件：m ≥2n-1
      • 可重排无阻塞条件：m≥n
  • TST网络
    • 网络结构
      • 32条双向时分复用线，每条时分复用线有32个时隙，编号相同的入线出线构成一条双向时分复用线
      • 第一级有32个T接线器，第二级是一个32x32的S接线器，第三级有32个T接线器
      • 一般情况下，第一级为输出控制，第二三级均为输入控制
    • 详细过程会在第三章描述
    • 反向路由的==半帧选择==
      • 交换网络一般建立双向通路，除了建立A→B的信息传输，还需要B→A的信息传输
      • 反相法指两个方向的内部时隙相差半个帧（帧指TST网络输入/输出的复用帧）
    • TST完全无阻塞条件内部时隙m=2x输入时隙n
    • 同发特性
      • 初级和次级T采用不同的控制方式
    • 为什么不用T-S或者S-T必须用TST呢
      • 要把HW0的TS0-TS31分别交换到HW0-HW31的TS0上、T-S网络就无法实现，只能用S-T网络
      • 要把HW0-HW31的TS0分别交换到HW0的TS0-TS31上，S-T网络就无法实现，只能用T-S网络
  • BANYAN网络 
    • 各种banyan子类
      • 具有多级结构的交换网络
      • 如果任何一条入线到出线的通路都经过了L级—L级banyan网络
      • L级banyan网络中如果构成网络的所有交换单元都相同则为规则banyan，否则为不规则banyan
      • 如果规则banyan中交换单元的入线=出线，则为矩形banyan网络
    • 实际使用基本都用2x2构成的矩形banyan网络
    • 交叉连接单元
      • 5种工作状态（一般只用直连和交换，称为两功能交换开关）
    • banyan网络的构造
      • 一级二级采用蝶式连接、二级三级采用子洗牌连接
    • banyan网络的6大特点
      • banyan是基于树形结构的
        • 每个输入端可以到达任何输出端
        • 构成了某一输入端为根，所有输出端为叶子结点的树形结构
      • banyan网络的级数
        • k=log₂N，每级N/2个交换单元
      • banyan网络的构成规律——小变大
        • 组成2nx2n的banyan网络要先准备两个nxn的网络和n个2x2的交换单元，第一个nxn的n个出端口分别接到n个交换单元的一号口，第二个nxn的n个出端口分别接到n个交换单元的二号口
      • banyan网络具有唯一路径特性
        • 任何一条入线和一条出线有且仅有一条通路
        • 证明：递归证明
      • banyan网络具有自选路由的特性
        • 给进入交换网络的信息加上选路标签，标签就是出号的二进制值，然后只要没经过一个交换单元都根据对应标签位的值进行选路，就可以到达对应的出端口
      • banyan网络是有内部阻塞的
        • 各入线和出线之间的单通路并非完全分离，会有公共的内部链路
      • 降低阻塞方法：
        • 增长型banyan网络
          • 增加分配级交换单元，减少内部阻塞
          • 每增加一级，通路数翻倍
          • 缺点：路由复杂，失去唯一路径和自选路特性。
        • 扩展型BANYAN 网络
          • 扩充了每条路的链路数目，扩展度d
        • 膨胀型BANYAN网络
          • 扩展度可以逐级增加的扩张型
          • 如第一级d=2，第二级d=3···
        • BATCHER-BANYAN（排序-BANYAN）网络
          • 如果将banyan输入的信息按照出线地址进行单调递增/递减排列，可以解决内部阻塞，因此可以先加入排序网络
          • 严格无阻塞、结构规整、自动选路。
          • 无法消除外部竞争
        • 多平面banyan网络
          • 多个banyan并接，想去哪个去那个就不会冲突了（也可以平均分）
          • 硬件复杂
        • 减少入线的信息量，加大入线缓存*
BENES网络

  • 著名的可重排无阻塞网络
  • 构造法
    • 两个banyan类网络背对背连接 ，然后把中间两级合并

程控数字交换机的硬件
  • 程控==数字==交换机的系统结构
    • 基本结构任由控制子系统和话路子系统组成，但是采用模块化分级控制结构方式
    • 控制子系统
      • 采用分级分散控制的方式，由中央级控制系统和用户级控制系统两级组成的
        • 用户级控制系统就是用户级CPU，一般负责对用户模块内所有用户线路进行监视扫描，控制用户级交换网络
        • 中央控制级由CPU、存储、OAM终端、外设、远端接口组成
        • 源端接口是控制子系统与集中操作维护中心、网管中心、计费中心等数据传送接口
      • 中央CPU负责系统资源分配、中央交换网络的控制、呼叫处理、信令处理、控制用户级CPU以及完成系统的操作、维护、管理等功能
    • 话路子系统
      • 由中央级交换网络、用户级交换网络、各种接口设备组成
      • 交换网络
        • 完成交换功能，实现不同线路端口话音交换
        • 数字程控交换机的交换网络是数字交换网络，采用T接线器或T+S接线器，按特点拓扑和控制方式完成时分复用信号的交换
      • 接口设备
        • 数字程控交换机与外围环境的接口
        • 完成外部信号与交换机内部信号的转换
        • 主要有用户电路、中继电路、信令收发设备
          • 用户电路是用户终端与交换机的接口，用户终端通过用户线连到交换机，每条用户线对应一套用户电路
          • 交换机与交换机的通信线路叫中继线，中继电路是连接中继线的接口，现在数字中继电路较多，模拟中继电路已经很少使用
          • 信令收发设备接受用户线、中继线信令的发送和接受
    • 模块化结构——由中央级模块和用户级模块组成
      • 中央级模块由中央级交换网络、信令接口、中继接口和中央级控制系统组成
      • 用户级模块指用户模块，由用户级交换网络、用户电路和用户CPU组成
        • 主要完成话务量集中（集中比2:1or4:1）这样可以提高链路的利用率
        • 用户级交换网络一般为单T交换网络，只有话务集中功能
        • 用户模块CPU负责用户线的扫描监视、用户级交换网络的控制
      • 远端用户模块是放置在远离中央且用户集中的地方，中央称为母局
        • 远端用户模块与母局之间采用PCM链路通过数字中继设备连接
        • 节省了用户线路的投资
        • 模拟信号转数字信号传输，改善了用户质量
        • 比用户模块的额外功能
          • 具有模块内的交换功能
          • 当与母局的PCM出现故障的时候，模块内部用户可以正常通话，至少保存24h计费信息，一旦与母局回复联系联系，可以将计费信息传至母局
  • 用户模块（用户电路+用户集中级）
    • 模块组成
      • 用户电路：每条用户线有一个用户电路，完成信号采集、动作驱动、话音传输等功能
      • 交换电路：用户级交换和话务集中（集中比为 2：1 或 4：1）
      • 收号器：接收用户的DTMF号码
      • 扫描存储器：接收各个用户电路的扫描信息，送用户处理机
      • 分配存储器：接收用户处理机对各个用户电路的指令，控制各用户电路继电器的动作
      • 用户处理机：与中央处理机协调控制用户电路和交换电路的工作
      • 通信电路（信号提取插入）：负责收发用户处理机与中央处理机之间通信的信息
      • 网络接口：负责把用户的话音和处理机来的消息合并在一起，送到中央级的数字交换网络
    • 用户电路（不要求）
    • 用户电路的基本功能：BORSCHT
      • B——馈电
        • 交换机通过用户线向用户终端提供通信的电源，由交换机的用户链路完成
        • 我国-48V/60V，国外-48V
      • O——过压保护
        • 严禁高压进入、破坏交换机的内部设备
        • 避雷装置保安器+过压保护
      • R——振铃控制
        • 25Hz交流电压为振铃电压
        • 通过继电器/高压电子器件向话机提供
      • S——监视
        • 监视用户线直流环路电流的通/断来区分
          • 用户话机的摘挂机
          • 用户话机发出的拨号脉冲
          • 投币、磁卡等话机的输入信号
      • C——编译码和滤波
        • 编译码器完成模拟信号和数字信号之间的转换（话音信号是模拟信号）
        • 滤波器避免模/数变严重信号取样产生的混叠失真和店员干扰
      • H——混合电路
        • 完成二/四线转换
        • 用户话机的模拟信号是二线双向，数字交换网PCM数字信号是四线单向
        • 平衡网络用于平衡用户线阻抗
      • T——测试
        • 配合外部测试设备对于用户线进行测试
  • 中继器/中继电路的基本功能（不要求电路）
    • 交换机和中继线的接口设备
    • 现在主要使用的是数字中继电路，用于与数字交换局或远端用户模块相连![[截屏2024-06-12 16.44.00.png|600]]
    • 主要功能
      • 码型变换
        • PCM线上使用的传输码型与交换网络内部的码型不同
          • PCM线上使用HDB ₃
          • 交换机内部使用NRZ
      • 帧同步
        • 从接受的数据流中搜索并识别到帧同步码，以确定一帧的开始，使得接收端的帧结构排列和发送端的一致，保证数字信息的正确接收
          • 帧同步码0011011在PCM偶帧的TS₀种
        • 帧同步中会有两个基本状态：帧同步状态和帧失步状态
          • 在给定帧同步码位上检测出帧同步码称为帧同步状态（该出现的位置出现了）
          • 连续三次检测到码字与帧同步码不相符则判定为帧失步（没对齐），这时系统会在奇帧的TS₀发出失步告警码通知对端局
          • 当连续两个偶帧检测到同步码，恢复到帧同步状态
      • 复帧同步（采用随路信令）
        • PCM的一个复帧由16个帧组成
        • 复帧同步是使接收端与发送端的复帧结构排列完全一致
        • 保证各路线路信令不错路

    • 复用器和分路器
      • PCM信号传输采用串行码，即一个时隙的8位码在一条线路上，T接线器的话音存储器字长一般为8位，写入和读出是以字长为单位进行的，8位码需要并行同时写入或读出
      • 因此话音信号进入交换网络前先要把串行码转换为并行码，当话音信号完成交换从交换网络输出时也要把并行码转换成串行码

        • 信号的串并变换
        • 多路低速信号进行时分复用变为告诉时分复用信号
      • 分路器功能
        • 信号的并串信号

      • ①处速率为2048kbit/s串行码
      • ②处速率为256kbit/s并行码
        • 一个时隙的8位D0～D7并行输出
      • ③处速率为1024kbit/s并行码
        • 4条（HW）变8条（Dᵢ），速率减半

        • k=jxn+i
        • 原理：变换后的顺序是：0号线路的0时隙、1号线路的0时隙···n-1号线路的0时隙，0号线路的1时隙···，如果把输出的每n个时隙看成一组，编号i的第j时隙前面有j组（从0开始），j组共jxn个时隙（n路轮流传一个时隙），轮到目标组之后，第i线路排在第i+1个，所以是第jxn+i+1个时隙，因为输出时隙也是0开始，所以是TSⱼₙ₊ᵢ

    • 控制系统的构成方式
      • ==体现在功能、资源和处理机上==
      • 集中控制：
        • 处理机对交换系统内==所有功能及资源统一控制==，多个处理器的每个都可以控制整个系统
        • 早期程控交换机多用此形式
        • 特点
          • 处理机直接控制所有功能的完成和资源的使用，因此控制关系简单，处理机间==通信接口简单==
          • 单个处理机上的==应用软件复杂、庞大==。
          • 处理器集中完成所有功能，一旦处理机系统故障，整个控制系统失效。==可靠性较低==
      • 分级分散控制：
        • 控制系统由多个处理机构成，==分别==完成不同功能并对不同资源实施控制
        • 现代程控交换机多用此形式
        • 分散控制又可以分
          • 全分散控制
            • 各个处理机之间独立工作，分别完成不同功能并对不同资源实施控制（一人一活）
            • 特点
              • 处理机之间不分等级，不存在控制与被控制关系。
              • 每台处理机只完成部分功能，要求各处理机协调配合完成整个系统功能，通信接口复杂。
              • 每台处理机应用软件只完成该处理机承担的功能，较为简单。
              • 可靠性较高，一个处理机崩坏不影响系统
              • 系统具有良好的扩充能力
          • 分级分散控制
            • 各处理器分别完成不同功能并对不同资源实施控制，处理器分等级
            • 特点
              • 处理机之间分等级，高级别处理机控制低级别处理机
              • 处理机之间通信接口较为集中，控制方式复杂，但比全分散简单
              • 各处理机上应用软件复杂程度适中
              • 控制系统的可靠性适中
            • 一般为2～3级

    • 功能分担：多个处理机分别完成同一话务的不同功能
      • 如用户CPU、中继CPU、信号音CPU、中央CPU分别完成不同的功能
    • 话务分担：多个处理机各自完成一部分话务功能
      • 如每个用户CPU之间分担一组用户
    • 冗余方式：为了提高可靠性配置备用机
      • 双机冗余配置：两套处理机系统，一个主用、一个备用
        • 同步方式：主备用机同步工作，同时执行指令并比较结果（同步双处理机工作系统）
        • 互助方式：主备用机负荷均分，分别承担一半话务负荷。一台机器故障，负荷全部转移到另一台机器上
        • 主备用方式：主机在线运行，备用机处于待机状态
          • 冷备用：不保存动态呼叫数据，故障切换时正在进行的呼叫会损失
          • 热备用：保存动态呼叫数据，故障切换时不会呼损
          • 一般采用热备用
      • N+m 备份：N 个处理机在线运行，m 个处理机处于备用状态
        • 一般采用N+1冗余

    • 时间表调度算法
      • 每次时间中断到来时，对时间计数器加一，根据时间计数器的值形成时间表行地址屏蔽表用于控制在该时刻程序是否被调用执行
      • 时间表的每一行代表一个时间单元，每一列为一个比特，代表一个程序（最下方）
      • 若在第 i 行第 j 列的比特位的值为“1”, 则表示在这个时刻该程序被调用；若为“0”则不被调用
      • 每次时间中断到来时，都要对时间计数器做加“1”操作，时间计数器的值形成了时间表的行地址
      • 程序地址表保存被调用程序的入口地址
      • 屏蔽表用于控制在该时刻该程序是否被调用执行，屏蔽表的每一位对应一个程序，如果某一位为“1”则表示该程序可执行，否则不执行
        • 屏蔽表提供了一种灵活控制程序调用的机制，不用频繁更改时间表（更改屏蔽表就可以修改表内内容）
      • 若时间中断周期为 10ms，则由上述表格结构的设计可知：
        • 拨号脉冲识别程序每隔 10ms 被调用执行。
        • 按键号码识别程序每隔 20ms 被调用执行。
        • 位间隔识别程序每隔 100ms 被调用执行。
        • 用户线扫描程序每隔 100ms 被调用执行。
        • 中继线扫描程序每隔 100ms 被调用执行。

MPLS的基本组成和原理
  • 简介
    • MPLS是一个可以在多种第二层媒质上进行标记交换的网络技术，结合了第二层的交换和第三层路由的特点
      • ==将第二层的基础设施和第三层的路由有机地结合起来==
    • 同时可以支持==多协议==，使多种网络的互连互通成为可能
    • 使用固定长度的标记表示转发信息，使用精确匹配算法进行转发判决，故称==标记交换==
  • 概念
    • LSP：Label Switched Path
      • 标记交换路径，类似虚电路。
    • FEC：Forward Equivalence Class
      • 前转等价类，灵活按照多种方式划分，相同 FEC 的包具有相同 Label，走相同 LSP
      • 走相同路径的一类包
    • LIB：Label Information Base
      • 标记信息库，保存转发 Labeled 分组所需要的信息，所有分组的路由
    • LER：Label Edge Router
      • 边缘路由器
      • 入：为每个 FEC 生成 Label，映射到 LSP 下一跳的标记，对入口IP分组进行分类，确定FEC，根据FEC查询LIB得到下一跳Label，将Label插入IP包头，从相应端口发送
      • 出：去掉 Label 还原成普通 IP 包继续转发
    • LSR：Label Switch Router
      • 维护 LIB、完成标记置换
  • 基本思想：边缘路由、核心交换
    • 面向连接
    • 控制平面与转发平面分离：便于采用新的路由协议和交换技术
      • 控制平面指处理MPLS的信令和路由协议，负责LSP的建立、拆除、维护、重建、与FEC对应等
      • 数据平面指利用标记进行数据传输、标记的格式和封装
    • 通过标记堆栈实现多层次的转发：提高可扩展性
MPLS交换原理
  • MPLS交换流程
    • 建立连接
      • 通过LDP等协议，在LSR和LER、LSR和LSR之间完成标记信息的分发，形成与FEC对应的LSP路径
    • 数据传输
      • 入口LER接收分组，完成第三层功能，判定分组所属的FEC，根据FEC的到下一跳Label，插入IP包头并从下一跳端口发送
      • LSR依据分组上的标记以及标记转发表通过交换单元对其进行转发，完成标记置换（提取旧Label装入新Label）
    • 拆除连接
      • 去掉封装的Label，按IP路由方式继续传送
  • LDP ——普通标记分配协议
    • 是MPLS网络中交换节点之间交互的协议、用于创建、维护、删除标记交换路径LSP
    • 标记分配过程
    • 三种标签分配方式
    • 标签操作
      • Push：标签入栈，进入子路由域内传输
      • Pop：标签出栈，回到父路由域传输。
      • Replace：置换标签，在同一路由域内传输
    • 下游按需标签分配时，标签分配（LDP）过程和数据传输过程（要看清哪边是源哪边是目的）
      • 第一步：标记请求Label Request
        • 这里A要了一个去D的一个去E的标签，B要了一个去D的，根据路由表发送标签请求
      • 第二步：标记绑定Label Mapping
        • D和E分别按照路由发了回去分配的标签
        • 发送的途中就会把对应路由表的标签填好
      • 第三步：标记应用
        • 来了一个包就会按照路由表标志的标签打好
        • 中间也按照标签进行置换和转发
        • 最后把标签拆掉继续发IP

第五章  信令系统

  • 信令是呼叫接续过程中的一种“通信语言”，用于协调动作、控制呼叫，是通信网中规范化的控制命令
    • 比如“摘机”、“挂机”、“忙音”等
  • 随路信令CAS（1号信令）
    • 信令和用户信息在同一通路上传送的信令
    • 通信建立时用户信息通路空闲、可用于传送信令
    • 特点
      • 共路性：信令和用户信息在同一通信信道传送
      • 相关性：信令和用户信息在时间位置上有相关
    • 缺点
      • 传送速度慢、信令容量小
      • 传递与呼叫无关的信令能力有限
  • 公共信道信令CCS（7号信令）
    • 信令通道和用户信息通道不具有时间位置的关联性，彼此相互独立
    • 特点
      • 分离性：信令和用户信息在各自通信信道传送
        • 速度快、容量大
      • 独立性：信令和用户信息不具有时间位置关联
NO7信令系统
  • 新型的采用最佳指令速率为64kbit/s的CCS
  • 信令网组成
    • 信令点（SP）和信令转接点（STP）
      • 是提供No. 7信令的通信网节点
        • 通常就是通信网的交换节点或处理节点，如：交换局、网管中心、数据库、智能网业务控制点等
      • 信令源点、宿点
      • 信令转接点（独立/综合）
    • 信令链路
      • 是连接各个信令点，传送信令的物理链路
        • 通常就是通信链路的一部分，可以是光纤、微波、PCM电缆线路等
      • 链路组
        • 一束信令链路构成一个“链路组”
        • 一个链路组常常包括所有并行的信令链路
  • 优点
    • 信令传送速度快
    • 信令容量大
    • 能完成与呼叫无关的信令传送，支持多种业务
    • 多路通信的信令在公共信令设备上传送，设备经济
  • 特点
    • 采用公共信道的方式，数据链路是速率为64kbit/s的双向数据通道
    • 采用分组传送的数据包方式
    • 必须对话路进行单独的导通校验
    • 必须设置备用设备来保证可靠性
  • No.7信令系统结构（协议栈）
    • 基本功能结构：
      • 由消息传递部分MTP和用户部分UP组成
      • 消息传递部分MTP
        • 在信令网中提供可靠的信令消息传送
        • MTP1:信令数据链路功能
          • 对应了OSI中的物理层
          • 可为信令提供一条64kbit/s的双向数据通路
          • 对应于PCM传输系统中的一个时隙
        • MTP2:信令链路功能
          • 对应OSI中的数据链路层
          • 保证信令点之间消息的可靠传送
          • 主要功能：信令单元（相当于帧）的定界、定位、差错检验和纠错和流量控制
        • MTP3:信令网功能
          • 对应OSI中的网络层
          • 消息处理功能
            • 保证将信令发送到相应的信令链路或用户部分，完成发送的选路和接收的分配/转发
            • 包括消息识别、消息分配和消息路由
          • 网管理功能
            • 控制信令路由和信令网结构，在信令网故障时提供网络结构的重新组合
            • 信令业务量管理、信令链路管理和信令路由
      • 用户部分UP
        • 第四功能级，主要功能是控制各种基本呼叫的建立和释放
        • 可以是
          • 电话用户部分TUP
            • 电话呼叫的建立、释放的信令流程、实现流程的消息和消息编码
          • 数据用户部分DUP
          • ISDN用户部分ISUP
        • TUP功能：
          • 规定电话呼叫的建立和释放的信令流程以及实现这些流程的消息和消息编码
          • 支持部分用户补充业务。提供电话呼叫的控制信令，完成电话呼叫续接和控制
          • 呼叫指令

            • SLB：市话忙
            • STB：长话忙
            • CGC：电路群拥塞
            • SEC：交换机拥塞
            • ANC：被叫应答、计费消息
            • CLF：前向释放
            • CBK：后向释放
            • RLG：正常呼叫结束时电路释放监护消息
          • 呼叫流程
      • 七号信令4级功能结构
    • 面向OSI七层协议的信令系统结构
      • MTP3和网络层的区别：MTP3不能跨网寻址、不能端到端信令传递、不能传递与电路无关的信令、不支持逻辑连接
      • SCCP：信令连接控制
        • 弥补了MTP3的不足，加强了消息传递的路由和寻址

        • 包括事物处理能力应用TCAP和中间业务部分ISP
      • MAP：移动通信应用部分

    • 信令单元的基本结构
      • 采用可变长的信令单元，若干个8位位组组成，有3种信令单元格式
        • 消息信令单元MSU
          • 用来传送第四级用户级信令消息或信令网管理消息
        • 链路状态单元LSSU
          • 链路启用或者链路故障时用来表示链路状态信息
        • 填充单元FISU
          • 用于链路空或者链路拥塞时填补位置
    • MTP2 处理：
      • 标志码F
      • 校验位CK
      • 前向序号FSN
      • 后向序号BSN
      • 前向指示比特FIB
      • 后向指示比特BIB
      • 长度标识语LI
    • MTP3 处理
      • 路由标记和业务标记8位位组SIO。

        • DPC：目的信令点编码

        • SLS：信令链路选择码
      • 路由标记供 MTP3 级进行消息寻址和路由。
    • 电话用户部分 TUP 处理
      • MSU 信令中 SIF 子段的
        • CIC：话路编码，标记话路时隙
        • H0H1：是标题码
        • 信令信息传递消息参数
  • 信令的工作方式
    • 根据通话电路和信令链的关系可以采用以下三种方式
      • 直联工作方式：
        • SP 和 SP 之间的信令消息通过直接相连的信令链路传送（链路专为两个端局服务）
      • 准直联工作方式：
        • SP 和 SP 之间的信令消息通过（STP 转发）预先指定的多条串接的信令链路传送。
        • 两段或者两段以上的信令链路
      • 非直连工作方式：
        • 不预先指定，随机转发，动态选择
        • 复杂，No.7 未采用
  • 我国的信令网结构
    • 长途网三极、本地网二级
    • No.7信令网是由信令点、信令转接点和信令链路组成
    • 三级信令网结构
      • 高等级信令转接点HSTP（省级）
      • 低等级信令转接点LSTP
      • 信令点SP，包括
        • 电话交换局
        • ISDN交换局
        • 业务交换点SSP
        • 业务控制点SCP
    • 连接方案
      • HSTP采用两个平行的A、B平面网，AB平面内部的HSTP网状连接，AB间成对的HSTP相连
        • 一个省一对HSTP，省间全连接
      • LSTP与HSTP和SP之间的连接方式为分区固定连接方式，每个LSTP通过信令链至少连接两个平面的一对HSTP，这两个HSTP采用负荷分担方式
        • 一个市一对LSTP，一个LSTP连接该省的一对HSTP
      • 每个SP至少连接两个STP（H/L）如果连接HSTP要连接AB成对的HSTP，两个STP采用负荷分担的方式
        • 每个SP连接该市的（其中）一对LSTP
      • 每个信令链路组中至少包括两条信令链路
      • 我国本地使用的汇接方式有来话汇接、去话汇接、来去话汇接、主辅汇接
    • 电话网和信令网对应关系
      • HSTP 设在 C1、C2 交换中心
      • C1 和 C2 长途局直接与 HSTP 相连
      • HSTP 汇接 C1、C2 及所属 LSTP 的信令
      • LSTP 设在 C3 交换中心，同一分信令区内的长途局应与本区内的每对 LSTP 相连
      • LSTP 汇接 C3、C4、C5 信令点的信令

SIP信令
  • 会话发起协议SIP是一种 IP 电话信令协议
  • 基本特点
    • **应用层控制协议，独立于底层协议。用于建立、修改和中止IP网上的双方或多方多媒体会话
    • 基于文本的消息编码，使用 UTF-8 字符集，易于读取和调试
    • 具有多个层次的可实现性，最小的实现非常简单、最完全的实现能够完成非常多的功能
    • 通过代理、重定向功能支持用户的移动性
    • 可与 RTP/RTCP、SDP、RTSP、DNS 等协议配合。易于实现和扩展
      • SDP描述多媒体会话
      • RTP/RTCP实时传输
  • SIP 系统中的各种设备
    • 用户代理UA
      • 终端用户设备
      • 可以发起呼叫请求也可以回应呼叫请求
    • 代理服务器ProxyServer
      • 接收 UA 的会话请求并查询 SIP 注册服务器，获取收件方UA的地址信息
      • 完成路由功能将SIP用户请求发送到下一跳代理或用户
    • 注册服务器RegisterServer
      • 接收用户的注册请求，记录终端的IP地址，记录用户的当前位置信息，是实现号码移动性的基础
      • 包含域中所有用户代理位置的数据库
    • 位置服务器LocationServer*
      • 配合代理服务器和重定向服务器提供可能的被叫用户的地址(或位置)
    • 重定向服务器RedirectServer*
      • 允许代理服务器将SIP会话邀请信息定向到外部域
      • 重定向服务器可以与注册服务器和代理服务器同在一个硬件上
  • SIP的网络布局
  • SIP的通话流程
    • 主叫找代理服务器
    • 代理服务器找重定向服务器
      • 重定向服务器查询位置服务器被叫用户地址（非SIP）
    • 代理服务器路由转发
    • 被叫返回回应
    • 话路建立

    • 呼叫准许；注册和接入管理（TCP/UDP）
      • 会话发起协议SIP
    • 呼叫建立（TCP/UDP）
      • 会话发起协议SIP
    • 媒体通道协商（TCP/UDP）
      • 会话描述协议SDP
    • 通道建立（TCP/UDP）
      • 会话描述协议SDP
    • 媒体流（UDP）
      • 实时传输协议RTP
  • SIP消息以及消息格式
    • SIP协议是一个基于文本的协议，包括请求和响应
      • 请求：INVITE、ACK、OPTIONS、BYE、CANCEL、REGISTER等
      • 响应：1XX、2XX、3XX、4XX、5XX、6XX等
    • 消息格式
    • 请求
      • 呼叫控制请求
        • ==INVITE== 发起呼叫，并对会话进行描述
          • ==携带主角SDP==
        • ==ACK== 主叫确认收到被叫发送的对INVITE的确认响应
        • ==BYE== 释放连接，由主叫方或者被叫方发出
        • ==CANCEL== 主叫取消呼叫，在连接建立起来之前发送
      • 注册请求
        • ==REGISTER== 在注册服务器上注册用户代理
      • 能力查询请求
        • ==OPTIONS== 查询服务器的能力
    • 响应
      • ==1XX==:通知服务器或代理正在执行处理，终端应该等待响应
        • 100:Trying
        • 180: Ringing
      • ==2XX==:请求成功
        • 200:OK
          • 携带被叫SDP
      • ==3XX== :重定向响应，终端应向新地址发起新请求
      • ==4XX== :请求失败，终端的请求被拒绝
      • ==5XX== :服务器内部错误造成请求不能被响应
      • ==6XX== :全局错误，所有未来的对该用户的请求都将失败
  • SIP完整消息结构
    • From：发起者AOR地址
    • To：目的端AOR地址
    • Call-ID：呼叫标识
    • CSeq：数据序号
    • Contact：响应时的直接消息路由（回信往这里发）
    • Content-Type/Length：消息体类型/长度

第六章  移动交换

移动通信的基本概念
  • 入网
    • 指移动设备（如手机）首次接入移动网络，成为该网络的一部分
    • GSM 中，MS 向 BSC 申请 SDCCH 或 TCH 信道进行通信的过程。GPRS 中，MS 向 SGSN 发送入网附着请求的过程
  • 鉴权
    • 鉴权是验证用户身份的过程，以确保只有合法用户可以接入和使用网络服务
    • 在GSM网络中，鉴权通常使用Ki密钥和RAND随机数通过A3算法生成响应值（SRES）来验证用户身份。如果用户身份被验证通过，用户即可获得服务访问权限
  • 切换
    • 将一个正处于呼叫状态中的 MS 转换到新的业务信道上的过程。通常由 MS 位置移动或小区业务负载均衡触发
    • 当用户在移动过程中信号质量下降时，网络会选择一个信号更强的小区，并将连接无缝切换过去。这可以是同一基站的不同小区间切换（小区内切换），也可以是不同基站间切换（小区间切换）
  • 漫游
    • 用户当前位置非本号码的 HLR 服务区
    • 当用户离开本地网络覆盖范围时，设备会自动搜索并连接到目标网络运营商。用户可以在异地或国外继续使用语音、短信和数据服务。漫游通常涉及两个运营商之间的协议和费用结算。
  • 位置更新
    • 位置更新是指移动设备向网络报告其位置变化的过程，以便网络能够正确路由呼叫和消息
    • MS 从一个位置区移动到另一个位置区，发现其存储器中的 LAI 位置区标识与接收到的 LAI 发生变化，便执行登记的过程
  • 网络附着
    • 网络附着是指移动设备连接到移动网络并注册以获得服务的过程
    • MS 附着在 GPRS 网络上，MS 和 SGSN 中记录该用户的路由信息，建立对于该用户 IMSI 的移动管理上下文（PDP）
GSM的基本网络结构及网元功能（2G）
  • GSM系统的网络结构
    • 主要分为 MS、BSS、NSS、OMC 四大部分
      • MS（移动台）
        • 移动客户设备部分，分手持机和车载台。包括移动终端ME、手机客户识别卡SIM
        • ME
          • 负责话音编码、信道编码、信息加密、信息调制解调、信息发射和接收
        • SIM
          • 存储各种信息比如用户识别码IMSI、PIN码、PUK码、鉴权加密信息、呼叫限制、缩位拨号等等
          • 与网络数据库中存储的信息一致
      • BSS（无线基站子系统）
        • 由 MSC 控制，是与 MS 通信的系统设备。主要完成无线发送接收和无线资源管理等功能。
        • BTS（基站收发信机）
          • 完==成无线传输、无线与有线的转换==、无线分集、无线信道加密。
        • BSC（基站控制器）
          • 连接 BTS 与 MSC、为 BTS和OMC的信息交换提供接口，具有控制一个或多个 BTS 的功能。
          • ==完成无线网路资源的管理、呼叫和通信链路的建立与拆除==、本控制区内 MS 越区切换的控制、小区配置数据管理、功率控制等。
      • NSS（交换网路子系统）
        • 完成==交换功能==、完成客户数据、移动性管理、安全性管理所需的==数据库功能==
        • MSC（移动交换中心）
          • 蜂窝通信网络的核心，对位于本 MSC 控制区内的移动用户进行通信控制和管理
          • 完成信道的管理和分配、呼叫的处理和控制、用户位置信息的登记与管理、越区切换和漫游的控制、用户号码和移动设备号码的登记和管理、服务类型的控制、用户鉴权
          • 为系统中连接其他 MSC 和其他公用通信网络（PSTN、ISDN、PDN）提供链路接口。
        • HLR（归属位置寄存器）
          • 存储本地用户位置信息的数据库。每个用户都必须在某个 HLR（相当于该用户的原籍）中登记。
          • 登记内容：
            • 永久性参数：ISDN 号码、移动设备号码 IMSI、接入优先级、预定业务类型、保密参数等。
            • 暂时性参数：用户当前所处位置的有关参数
          • HLR 的目的是：
            • 即使用户漫游到该 HLR 服务区外，HLR 也要登记漫游区域传送来的位置信息。
            • 保证呼叫一个未知位置的移动用户时，均可由该移动用户的 HLR 获知它当前处于哪个区域，从而建立通信链路。
      • VLR（访问位置寄存器）
        • 存储来访用户位置信息的数据库。一个 VLR 可以为一个或多个相邻 MSC 服务。
        • 当移动用户漫游到新的 MSC，向对应的 VLR 申请登记。
        • VLR 从该用户的 HLR 查询参数，给用户分配一个新的漫游号码（MSRN），通知其 HLR 修改用户位置信息。为其他用户呼叫此用户提供路由信息。
        • 移动用户离开此 VLR 服务区时，HLR 接收到新的 VLR 发来的消息，通知旧的 VLR 删除此用户的位置信息。
      • AUC（鉴权中心）
        • 可靠地识别用户的身份，只允许有权用户接入网络获得服务。
        • 用户管理
      • EIR（设备标志寄存器）
        • 存储移动台设备参数的数据库
        • 识别用户的 IMEI，对移动设备进行鉴别和监视，拒绝非法移动台入网。
        • 识别设备
      • SMS-SC（短信息业务中心）
        • 提供点对点短信服务和广播式公共信息服务。
    • 维护操作子系统（OMC）
      • 对整个 GSM 网路内各种部件进行功能监视、状态报告、故障诊断、设备管理。包括系统的自检与报警、备用设备的激活、系统的故障诊断与处理、话务量的统计和计费数据的记录与传递、各种资料的收集分析与显示等。

GPRS 网络结构、网元功能（2.5G）
  • 概述
    • 在电路域的基础上新增了分组域
      • GPRS由GSM升级而成、网络结构和接口基本保持不变
      • 新增PCU、SGSN、GGSN等分组域网元
    • 逻辑信道新增分组数据信道PDCH
      • 采用四种新的信道编码，并支持多时隙（8个）传输方式
        • CS-1: 9.05kbps
        • CS-2: 13.4kbps
        • CS-3: 15.6kbps
        • CS-4: 21.4kbps
    • 采用动态资源分配
      • GSM采用电路方式，提供9.6/14.4kbps的数据业务，但一个用户只能分配一个信道
      • GPRS按需动态占用资源，速率可达171.2kbps
    • 采用流量+时间计费方式
  • GPRS系统的网络结构
    • 新增PCU、SGSN、BG、GGSN
    • PCU
      • 在 BSC 中新增的设备，属于分组域。负责移动分组数据的组装和拆解。
    • SGSN
      • 功能类似 MSC/VLR。
      • 记录MS当前位置信息，进行==鉴权==、==移动性管理==、==路由选择==
      • ==在 MS 和 GGSN 之间完成移动分组数据的发送和接收==
    • GGSN
      • 网关，功能类似 GMSC。
      • ==可以把 GPRS 分组数据包进行协议转换==，提供与多种不同数据网络的互联。
      • ==为 MS 动态分配 IP 地址==，或接入 DHCP 服务器来实现动态分配 IP 地址。
  • 电路域和分组域
    • 电路域：（原 GSM 部分）BTS、BSC、MSC/VLR，HLR、AUC 等。
    • 分组域：PCU、SGSN、GGSN、BG（？）、骨干网、ISP 网等
  • GPRS 分组域数据传输与控制（了解手机从开机到可以进行数据传输的过程）
    • 分组域用户平面协议栈（数据通道）
      • BSSGP
        • 在BSC（PCU）和SGSN之间
        • 提供虚电路NS-VC，透明的在SGSN和PCU间传输信号和数据
        • 管理SGSN和PCU之间的包交换链路资源
      • GTP
        • 采用隧道技术，允许多协议包在GPRS骨干网和GGSN之间的传输
        • 对信号信息和应用数据负责（不透明）
        • 基于IP协议栈，在OSI传输层使用UDP
        • 同时也对付费信息的传输负责
    • 分组域控制信令流程
      • MS的状态
        • IDLE
          • 用户未入网附着（Attach）在GPRS网络上，在SGSN上没有建立移动管理上下文（Context）
          • MS无法进行数据传输，也不能接收GPRS寻呼消息
        • STANDBY
          • 用户已入网附着在GPRS网络上，在SGSN上已经建立移动管理上下文（用户仅定位在位置区）
          • MS可以接收GPRS寻呼消息，但不能传输数据
        • READY
          • 与STANDBY状态相比，在SGSN上移动管理上下文中用户已定位在小区（Cell）
          • MS可以进行数据传送
        • 要进行数据通信，需要进行MS附着和PDP（Packet Data Protocol）上下文的激活
      • 附着过程
      • PDP上下文激活
        • PDP上下文保存了用户面进行隧道转发的所有信息，包括GGSN的用户面IP地址、隧道标识和QoS等
        • 激活过程用于激活IP协议，使MS与GGSN建立一条逻辑通路，进行数据传输

5G 通信
  • 5G业务需求：

      • 每平方公里一百万的连接数密度

    • 高频段高带宽传输：毫米波通信。

    • 新型多天线传输：
      • 手机中的毫米级多天线阵列与基站形成很多对的信道并行通信，

      • 大规模 MIMO，基站使用上百根天线，波束窄，==指向性传输==，高增益，抗干扰，提高频谱效率

    • 多种多址接入方案

SDN——软件定义网络
  • SDN定义
    • 一种新兴的基于软件的网络架构及技术
    • 具有松耦合的控制平面与数据平面、支持集中化的网络状态控制、实现底层网络设施对上层应用的透明，上层应用可以通过开放式API对网络进行控制
    • 传统网络设备紧耦合的网络架构被分拆成==应用、控制、转发三层分离的架构==（控制功能被转移到了服务器）
    • 使得网络的自动化管理和控制能力获得空前提升，能够有效解决当前网络系统所面临的==资源规模扩展受限、组网灵活性差、难以快速满足业务需求==等问题
  • 与传统IP网络的区别——控制平面、 转发平面分离
    • SDN的思路是将网络设备的控制平面集中上收到Controller
      • 网络设备上只保留转发平面(转发表项);
      • 通过Controller实现网络统一部署和网络自动化
    • SDN 的核心诉求
      • 让软件应用参与到网络控制中并起到主导作用，而不是让各种固定模式的协议来控制网络。
  • 转发、控制、应用的三层架构与南向北向接口
    • 三层架构
      • 应用平面：
        • 各类基于SDN的网络应用，通过开放式API完成对网络的控制
        • 不同的应用逻辑通过控制层开放的 API 管理能力控制设备的报文转发功能。（网络资源统一管理）
      • 控制层：
        • 由 SDN 控制软件组成，与转发层可通过 OpenFlow 协议通信。（设备管理和拓扑、流表控制和下发）
        • NBI代理：为第三方提供易用的北向接口，实现对网络的逻辑管理
        • ==控制逻辑：负责运行控制逻辑策略，维护全网视图==
        • CDPI驱动：控制器将全网视图抽象成网络服务，通过访问CDPI代理将转发规则发送到网络设备，具体功能包括==链路发现、拓扑管理、策略指定、表项下发==
      • 转发层（基础设施层）
        • 由转发设备组成（流表处理、数据转发）
        • 各网络元素之间由不同规则形成的SDN网络数据通路形成连接
        • 可以忽略控制逻辑的实现，==全力关注基于表项的数据处理==
      • 三大接口
        • 北向接口
          • 提供给其他厂家或运营商进行接入和管理的接口
          • 通过控制器向上层业务应用开放的接口，使得业务应用能够便利地调用底层的网络资源和能力
          • 直接为业务应用服务的，因此其设计需要密切联系业务应用需求，具有多样化的特征
          • 开放性、便捷性、灵活性
          • 能以软件编程的形式调用各种网络资源
          • 缺少业界公认的标准，主流实现方式为REST API
        • 南向接口
          • 设备控制信令，控制设备的转发行为。可用标准化 OpenFlow 协议。
        • 东西向接口
          • SDN设备之间的接口，多个设备的控制平面之间如何协同工作

  • Openflow 基本概念：将转发面设备抽象为一个由多级流表（Flow Table）驱动的转发模型。
    • OpenFlow交换机由安全通道、流表、OpenFlow协议三个部分组成*
    • OpenFlow交换机
      • OpenFlow交换机只包含转发功能，只需要按照流表的规则（由控制器下发）转发数据包，而控制器可以同时控制多台OpenFlow交换机
    • 安全通道
      • 安全通道（Secure Channel）是连接交换机与控制器的接口。通过该接口，控制器可以配置、管理交换机，接收来自交换机的事件和发送数据包给交换机
    • 流表
      • 在一段时间内，经过同一个网络的一系列具有相同属性的顺序发送的报文集合
      • 由若干条流表项组成
        • 流表项(Flow Entry)：流表的最小单位，每条流表项对应网络传输的一条流。根据OpenFlow规范，流表项组成如下：
          • 头字段：包含了用于匹配流（数据包）的字段信息，涵盖了L2-L4的各类协议信息
          • 计数器：用于统计所匹配流的各种信息，如包数、字节数
          • 动作：描述了对所匹配的数据包需要采取的操作
            • Forward：将数据包转发到指定口
            • Drop：丢弃数据包
            • Enqueue（可选）：将数据包发送至队列，用于QoS控制
            • Modifiy-Field（可选）：修改数据包的协议字段
      • OpenFlow交换机中可以有多个流表，每个流表可包含多个流表项
    • OpenFlow协议
  • 转发面抽象成多级流水线，
    • 每个节拍匹配关键字、操作指令集。
    • 和目前的SwitchASIC/NP模型类似
  • 转发面的行为
    • 根据转发表和报文头决定下一跳及新的报文格式。
    • 而SDN控制器就是控制转发表，并把转发面模型和控制协议标准化——OpenFlow
  • 流表的结构和作用，与路由表的区别
    • 流表是 OpenFlow 对网络设备的数据转发功能的抽象，表项包括了网络各个层次的配置信息。
    • 传统的路由表无法更改、不可编程。
